CONVENTIONS

Titres
------

A l'intérieur d'un chapitre, on utilisera des titres de niveau 2 ou supérieur.

Noms de logiciels, packages et fonctions
----------------------------------------

Logiciel : le nom sera indiqué en gras.
Exemple : **Word**

Package R : le nom sera indiqué en gras, dans une balise strong avec un attribut data-type égal à "package".
Exemple : <strong data-type="package">questionr</strong>

Fonction : le nom sera indiqué dans une balise code avec un attribut data-type égal à "fonction".
Le package dont est issu cette fonction sera précisé dans un attribut data-package.
Le cas échéant (pour une méthode par exemple), on précisera dans un attribut data-rdocumentation le nom
utilisé sur http://www.rdocumentation.org
Exemples :
<code data-type="fonction" data-package="questionr">freq</code>
la méthode <code data-type="fonction" data-package="prevR" data-rdocumentation="summary-prevR-method">summary</code>
pour les objets de la classe <code data-type="classe" data-package="prevR" data-rdocumentation="prevR-class">prevR</code>

Nom de variable : en italique

Argument d'une fonction :  balise code ou ``
Exemple : l'argument `object`

Valeur manquante : `NA`

Encadrés
--------

Les encadrés sont précisés avec une balise div et la classe CSS appropriée. Les différentes classes disponibles sont :

- info (bleu)
- note (gris)
- astuce (vert)
- important (jaune)
- attention (rouge)
